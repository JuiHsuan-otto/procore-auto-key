- 官網是唯一 canonical；Blogger、Threads、Google Business Profile 預設只產生草稿與回鏈素材，未經明確點名不得公開。
- GBP / My Business API 自動化路徑已由使用者決定停用；不要重試登入、OAuth、API 申請或發文，除非日後重新明確授權。
- 聯絡資訊固定：電話 `0909277670`、LINE `@420gknem`。
- 新文章建立後必須執行 `publish_tool.py`，同步 blog、cases、首頁最新案例與 sitemap；再執行 `python scripts/verify_site.py`。
- 案例內容去識別化，不公開車牌、VIN、姓名、完整地址、證件、設備型號或可被同業/惡意者利用的技術流程。使用者提供照片預設可公開；只在有 GPS metadata 時移除 metadata，不擅自裁切或改圖。
- Git 部署禁止 force push；網路或權限失敗時保留 commit 並如實回報，不能宣稱已上線。
- 2026-02-27 整合修復已建立本地提交；當時 GitHub HTTPS 受代理 502／逾時阻擋，需後續重試並確認遠端 HEAD。
- `/rescue-request` 是純前端救援需求整理工具；不得加入後端、儲存、追蹤或外部 API。
- `/service-areas` 的行政區查詢同樣必須維持純前端；分級僅依官網已列範圍提供保守指引，不得保證承接、到場或時效，僅可安全預填行政區至 `/rescue-request`。
